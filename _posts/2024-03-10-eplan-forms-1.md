## Overview

This repository documents the standard report and form structure of an **EPLAN Electric P8** project. EPLAN Electric P8 is a professional electrical CAD/CAE platform used for planning, documenting, and managing electrical and instrumentation engineering projects across industries including manufacturing, process, and energy.

The JSON file (`eplan_p8_project_structure.json`) was extracted from the EPLAN P8 project tree and enriched with:

- Form file extensions and internal type identifiers
- Purpose and description of each report folder
- Practical usage guidance per engineering phase
- Real-world examples for each form type
- Applicable IEC/ISA standards

## Project Tree Structure

## Form Descriptions

### 1. Cable Diagram `.f04`
Graphical representation of each cable showing routing, conductor assignment, type, cross-section, and shield. Used when documenting cable runs between panels, machines, or field devices.

**Example:** Cable W1 — type NYCWY 3×1.5mm², 15m, from terminal X1:1 (MCC) to terminal X2:3 (Motor starter cabinet).

---

### 2. Cable Overview `.f04`
Tabular summary of all cables with key attributes. Primarily used for procurement planning and installation scheduling.

**Example:** W1 (15m, NYCWY 3×1.5), W2 (8m, H05VV-F 2×1.0), W3 (22m, ÖLFLEX 5G2.5) — listed with source/destination terminals.

---

### 3. Connection List `.f17`
Point-to-point wiring list showing every connection in the project: from-terminal, to-terminal, wire color, cross-section, and potential. The primary document used by panel wiring technicians.

**Example:** From=X1:1 → To=K1:A1 | Wire: BK 1.5mm² | Potential: L1

---

### 4. Device Tag List `.f06`
Structured list of all device tags (DT) with article numbers, descriptions, quantities, and functions. Includes fluid variants for pneumatic/hydraulic components.

**Example:** Tag `=GB1+ET1-K1` → Siemens Contactor 3RT2026-1AP00, Qty: 1, Function: Motor starter.

---

### 5. P&I Diagram: Piping Overview `.f30`
Piping and Instrumentation Diagram per IEC 10628 / ISA 5.1. Shows process flow, equipment, and instrumentation relationships.

**Example:** Pump P-101, Flow transmitter FT-101, Control valve FCV-101, connected to DCS via 4-20mA signals.

---

### 6. PLC Diagram `.f22`
Auto-generated PLC I/O wiring diagrams. "SPS" is the German abbreviation for PLC (*Speicherprogrammierbare Steuerung*). Links field signals to I/O card channels and terminal addresses.

**Example:** Siemens S7-1500, DI card I0.0 = "Emergency Stop S1" → Terminal X10:1 → PLC slot 2, pin 1.

---

### 7. Pre-planning: Pipe Class Overview `.f31`
Pipe specification sheets used in FEED phase. Documents pressure class, material, temperature range, and applicable standards before detailed design.

**Example:** Class A1B — Carbon steel, PN40, −20°C to +300°C, used for steam utility lines.

---

### 8. Pre-planning: Planning Object Overview `.f32`
High-level functional planning objects used for top-down engineering. Covers PCT control loops, cable placeholders, and device assignments before detailed schematics exist.

**Example:** PCT Loop TIC-101 — Temperature controller with TT-101 (transmitter) and TCV-101 (control valve) linked to DCS.

---

### 9. Pre-planning: Planning Object Plan `.f33` — Loops
Loop diagrams showing full signal path from field device through junction box, marshalling, to control system I/O. Mandatory for FAT/SAT testing per ISA 5.4.

**Example:** TT-101 (4-20mA HART) → JB-05 terminal 3/4 → MC-01 marshalling → AI card slot 3 ch.7 → DCS tag TIC101_PV.

---

### 10. Pre-planning: Substance Overview `.f34`
Documents process media (fluids/gases) including physical properties, hazard classification, and ATEX zone. Drives material selection and safety design.

**Example:** Natural Gas — Phase: Gas, 10 bar, ATEX Zone 1, Hazard class II B T3.

---

### 11. Structure Identifier Overview `.f08`
Lists all reference designations used in the project per IEC 81346. Explains the `=`, `+`, `-` tagging hierarchy.

| Symbol | Meaning | Example |
|--------|---------|---------|
| `=` | Functional assignment | `=GB1` (Drive system) |
| `+` | Installation location | `+ET1` (Control cabinet 1) |
| `-` | Device/component | `-K1` (Contactor 1) |

**Full tag:** `=GB1+ET1-K1`

---

### 12. Summarized Parts List `.f05`
Consolidated BOM with summed quantities for identical parts. The definitive procurement document.

**Example:** 3RT2026-1AP00 — Siemens Contactor 11kW, Qty: 5, Unit: €45.00, Total: €225.00.

---

### 13. Table of Contents `.f01`
Auto-generated document index with page numbers, descriptions, and page types. Required for all formal documentation deliverables.

**Example:** Page 1: Cover sheet | Pages 2–15: Circuit diagrams | Pages 16–18: Terminal diagrams | Page 19: Parts list.

---

### 14. Terminal-Strip Overview `.f10`
Graphical/tabular layout of terminal strips showing terminal order, bridge groups, cross-connections, and conductor assignments. Used by panel builders for physical assembly.

**Example:** Strip X1 — T1: PE (GN/YE), T2: L1 → Motor M1 U1 (BN, 2.5mm²), T3: L2 → Motor M1 V1 (BK).

---

### 15. Terminal Diagram `.f11`
Detailed wiring diagram per terminal showing internal wiring and external cable conductor assignments. Used for complex marshalling panels and junction boxes.

**Example:** X1:5 — Internal: from -K3:14 (NC contact) | External: Cable W5 conductor 3 (BU, 0.75mm²) → Field solenoid -Y1.

---

### 16. Title Page / Cover Sheet `.f00`
Front matter of the documentation package. Includes project title block, revision history, RoHS/REACH material declarations, and Lockout-Tagout safety instructions.

**Subforms:**
- **Cover sheet** — Project metadata, drawn/approved by, revision table
- **Material regulations** — RoHS / REACH compliance declaration
- **Safety instructions** — IEC 60364 / VDE electrical safety notes, LOTO procedures

## EPLAN P8 Technical Notes

### Form File Locations
```
%APPDATA%\EPLAN\Electric P8\{version}\Forms\
```

### Generating Reports
```
Utilities → Reports → Generate
```
or use the **Edit Reports** dialog to configure which forms are included per project.

### Customizing Forms
Open the **Form Editor** from within EPLAN P8 to modify the graphical layout, properties displayed, and filter criteria of any form.
